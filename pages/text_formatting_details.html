Text Formatting Tags

HTML elements used to format and structure text content

  • Back to Main Reference

About Text Formatting Tags

Text formatting tags allow you to control the appearance, structure, and semantic meaning of text on your webpages. These elements are essential for creating readable content, establishing visual hierarchy, and improving accessibility.

Using the right text formatting elements not only enhances the visual presentation of your content but also improves search engine optimization (SEO) and accessibility for screen readers and other assistive technologies.

Heading Tags

<h1> to <h6>

Heading tags define HTML headings, with <h1> being the highest level (most important) and <h6> the lowest level (least important).

Example:

<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Section Heading</h3>
<h4>Sub-section Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Best practices for headings:

  • Use only one <h1> element per page, typically for the main title
  • Follow a hierarchical structure (don't skip levels like going from h2 directly to h4)
  • Keep headings concise and descriptive
  • Use headings to create a logical document outline

Paragraph and Line Break Tags

<p>

Defines a paragraph of text. Browsers automatically add margin space before and after paragraphs.

Example: <p>This is a paragraph of text.</p>

The paragraph element is one of the most commonly used text containers in HTML. It represents a logical paragraph of text.

<br>

Inserts a single line break. It's an empty element (has no closing tag).

Example: This text contains<br>a line break.

Use the <br> element sparingly. It's appropriate for addresses, poetry, or other content where line breaks are meaningful. Don't use it just to create visual spacing (use CSS margins/padding instead).

<hr>

Creates a horizontal rule (line) that acts as a divider between sections. It's an empty element.

Example:

<p>This is the first section.</p>
<hr>
<p>This is the second section.</p>

The <hr> element represents a thematic break between content. In HTML5, it has semantic meaning rather than just being a visual divider.

Text Emphasis and Highlighting

<strong>

Defines important text that is typically displayed in bold. It has semantic meaning indicating strong importance.

Example: This is <strong>very important</strong> text.

The <strong> element is preferred over the <b> element when the text has semantic importance, not just visual emphasis.

<em>

Defines emphasized text that is typically displayed in italic. It has semantic meaning indicating emphasis.

Example: This is <em>emphasized</em> text.

The <em> element is preferred over the <i> element when the text has semantic emphasis, not just visual styling.

<mark>

Defines text that should be highlighted or marked for reference. Typically displayed with a yellow background.

Example: Search results containing <mark>highlighted terms</mark>

The <mark> element is particularly useful for highlighting search terms, or marking important parts of text for reference.

<b>

Represents text that should be stylistically bold but doesn't have extra importance.

Example: The concert will be held on <b>July 15th</b> at 8 PM.

Use <b> when there is no added importance to the text but you want to stylistically bold it. Otherwise, prefer <strong>.

<i>

Represents text that is set off from the normal prose, such as foreign words, technical terms, or thoughts.

Example: The Latin phrase <i>carpe diem</i> means "seize the day".

Use <i> for text that would traditionally be italicized: foreign words, technical terms, thoughts, etc. For emphasis, use <em> instead.

Subscript, Superscript, and Other Text Formatting

<sub> and <sup>

<sub> defines subscript text and <sup> defines superscript text.

Example: H<sub>2</sub>O is water and 2<sup>3</sup> equals 8

Subscript and superscript are commonly used in mathematical formulas, chemical formulas, and footnotes.

<small>

Represents side-comments and small print, like copyright and legal text.

Example: <small>Copyright 2025. All rights reserved.</small>

The <small> element doesn't just make text smaller; it has semantic meaning for less important text.

<del> and <ins>

<del> represents deleted text and <ins> represents inserted text, often used to show document changes.

Example: This is <del>deleted</del> <ins>inserted</ins> text.

These elements are useful for showing edits or changes to documents and content.

Block Quotations and Citations

<blockquote>

Defines a section that is quoted from another source.

Example:

<blockquote>
  <p>The only thing we have to fear is fear itself.</p>
  <footer>—Franklin D. Roosevelt</footer>
</blockquote>

The <blockquote> element can include the cite attribute to specify the source URL of the quote.

<q>

Defines a short inline quotation. Browsers typically add quotation marks around the content.

Example: As Shakespeare wrote, <q>All the world's a stage</q>.

Use <q> for short inline quotes and <blockquote> for longer, block-level quotations.

<cite>

Defines the title of a creative work (e.g., a book, a poem, a song, a movie).

Example: My favorite book is <cite>The Great Gatsby</cite> by F. Scott Fitzgerald.

The <cite> element is typically rendered in italic text by browsers.

Preformatted Text

<pre>

Defines preformatted text that preserves both spaces and line breaks. Typically displayed in a monospaced font.

Example:

<pre>
function greet() {
  console.log("Hello, world!");
}
</pre>

The <pre> element is useful for displaying code, ASCII art, or any text where whitespace formatting is important.

<code>

Represents a fragment of computer code. Often used within a <pre> element for code blocks.

Example: The HTML <code>&lt;div&gt;</code> element is a container.

When showing HTML code examples, remember to escape the angle brackets (< becomes &lt; and > becomes &gt;).

Text Formatting Best Practices

  • Use semantic elements that describe the meaning of content, not just its appearance
  • Follow proper document structure with headings in hierarchical order
  • Use <strong> and <em> for semantic emphasis rather than <b> and <i> when appropriate
  • Avoid using <br> tags just for spacing (use CSS instead)
  • Use the appropriate quotation elements (<blockquote> and <q>) for quoted content
  • Keep paragraphs concise and focused on a single idea
  • Use preformatted text (<pre>) when whitespace and formatting matter
  • Consider accessibility implications when formatting text

Common Use Cases

Article or Blog Post

<article>
  <h1>Main Article Title</h1>
  <p>Introductory paragraph with <strong>important points</strong> and <em>emphasized content</em>.</p>
  
  <h2>First Section</h2>
  <p>Section content with a <q>brief quote</q> included inline.</p>
  
  <blockquote>
    <p>This is an extended quotation from another source that's displayed as a block.</p>
    <footer>—Author Name, <cite>Source Title</cite></footer>
  </blockquote>
  
  <h2>Technical Section</h2>
  <p>Some technical content with code example:</p>
  <pre><code>
function exampleCode() {
  return "Hello World";
}
  </code></pre>
  
  <p>Final thoughts and conclusion.</p>
</article>
