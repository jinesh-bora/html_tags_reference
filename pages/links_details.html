Link and Anchor Tags

Creating hyperlinks to connect web content

About HTML Links

Links are the fundamental building blocks of the World Wide Web, allowing users to navigate between pages and resources. The HTML <a> (anchor) element creates hyperlinks to other web pages, files, locations within the same page, email addresses, or any other URL.

Links are crucial for creating an interconnected, navigable web experience and are one of the most commonly used HTML elements.

The Anchor Element

<a>

Creates a hyperlink to web pages, files, email addresses, locations within the same page, or anything else a URL can address.

Basic syntax: <a href="destination">Link text</a>

The href attribute specifies the destination of the link, while the content between the opening and closing tags becomes the clickable text that appears on the page.

Link Types and Examples

External Links

Links to pages on other websites.

Example:

<a href="https://www.example.com">Visit Example.com</a>
<a href="https://www.example.com" target="_blank">Open in new tab</a>

Best practice: For external links that open in a new tab, add rel="noopener noreferrer" for security:

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Link text</a>

Internal Links

Links to other pages within the same website.

Example:

<a href="about.html">About Us</a>
<a href="/products/index.html">Products</a>
<a href="../contact.html">Contact</a>

Note: The path in the href attribute can be relative or absolute. In the examples above:

  • about.html - A file in the same directory
  • /products/index.html - An absolute path from the root directory
  • ../contact.html - A file in the parent directory

Anchor Links (Fragment Identifiers)

Links to specific sections within the same page.

Example:

<!-- Creating the target section with an id -->
<section id="section1">
  <h2>Section 1</h2>
  <p>Content for section 1...</p>
</section>

<!-- Creating a link to the section -->
<a href="#section1">Jump to Section 1</a>

You can also link to a specific section on another page:

<a href="page.html#section1">Jump to Section 1 on another page</a>

Email Links

Opens the user's default email client with a pre-addressed email.

Example:

<a href="mailto:info@example.com">Email Us</a>

<!-- With subject and body -->
<a href="mailto:info@example.com?subject=Website%20Inquiry&body=Hello%2C%20I%20have%20a%20question%20about...">
  Email with pre-filled subject and body
</a>

Note: For spaces and special characters in the subject and body, use URL encoding (e.g., space = %20, comma = %2C).

Phone Links

For mobile devices, creates a clickable phone number that can initiate a call.

Example:

<a href="tel:+12345678900">Call Us: (234) 567-8900</a>

The tel: protocol is especially useful for mobile users who can tap to call directly.

Download Links

Links that prompt the user to download a file rather than navigate to it.

Example:

<a href="files/document.pdf" download>Download PDF</a>

<!-- With a custom filename -->
<a href="files/document.pdf" download="my-document.pdf">Download PDF with custom filename</a>

The download attribute specifies that the target will be downloaded when a user clicks on the link.

Link Attributes

href

Specifies the URL of the page the link goes to. This is the most important attribute of the anchor element.

Example: <a href="https://www.example.com">Example</a>

The href attribute can contain:

  • Absolute URLs (https://example.com/page.html)
  • Relative URLs (page.html, ./page.html, ../page.html)
  • Anchor/fragment identifiers (#section-id)
  • Other protocols (mailto:, tel:, ftp:)
  • JavaScript (javascript:void(0)) - though this should generally be avoided in favor of proper event handlers

target

Specifies where to open the linked document.

Example: <a href="https://www.example.com" target="_blank">Example</a>

Values:

  • _self: Default. Opens in the same frame/tab
  • _blank: Opens in a new tab or window
  • _parent: Opens in the parent frame
  • _top: Opens in the full browser window, canceling all frames
  • framename: Opens in a named frame

rel

Specifies the relationship between the current document and the linked document.

Example: <a href="https://www.example.com" rel="nofollow">Example</a>

Common values:

  • nofollow: Indicates that the link should not be followed by search engines
  • noopener: Prevents the new page from accessing the window.opener property (security measure for target="_blank")
  • noreferrer: Prevents passing the referrer information to the linked page
  • external: Indicates that the link leads to an external site
  • author, help, license, prev, next: Describe the relationship to the linked document

download

Specifies that the target will be downloaded when a user clicks on the link.

Example: <a href="file.pdf" download="filename.pdf">Download</a>

The value of the attribute will be the name of the downloaded file. If no value is specified, the original filename is used.

Note: This attribute only works for same-origin URLs for security reasons.

title

Provides additional information about the link, often displayed as a tooltip when hovering over the link.

Example: <a href="page.html" title="Learn more about our services">Services</a>

The title attribute can improve accessibility and user experience by providing context about the link destination.

Link Content

Links can contain more than just text. They can wrap around:

  • Text
  • Images
  • Multiple elements
  • Block-level elements (in HTML5)

Examples:

<!-- Link with an image -->
<a href="index.html">
  <img src="logo.png" alt="Company Logo">
</a>

<!-- Link with text and image -->
<a href="products.html">
  <img src="product-icon.png" alt="">
  View Our Products
</a>

<!-- Link containing a heading and paragraph (HTML5) -->
<a href="article.html">
  <h3>Article Title</h3>
  <p>Short excerpt from the article...</p>
</a>

Accessibility Considerations

  • Descriptive link text: Use meaningful text that describes the destination, not generic phrases like "click here" or "read more"
  • Title attributes: Provide additional context when the link text alone might be ambiguous
  • Skip navigation: Include a "skip to main content" link at the beginning of pages with extensive navigation
  • Visual indicators: Ensure links are visually distinguishable from regular text (typically underlined by default)
  • Keyboard navigation: Make sure links can be accessed via keyboard (they should have focus states)
  • ARIA attributes: When needed, use aria-label or aria-labelledby to provide more context for screen readers

Example of accessible link practices:

<!-- Poor accessibility -->
<a href="products.html">Click here</a> to view our products.

<!-- Better accessibility -->
<a href="products.html">View our product catalog</a>

<!-- With additional context via title -->
<a href="products.html" title="View our complete product catalog with prices">
  View our product catalog
</a>

<!-- Skip navigation link -->
<a href="#main-content" class="skip-link">Skip to main content</a>

Best Practices for Links

  • Use descriptive, concise link text that makes sense out of context
  • Avoid opening too many links in new tabs (target="_blank") unless it benefits the user experience
  • Always add rel="noopener noreferrer" when using target="_blank" for security reasons
  • Make link text descriptive of its destination (not "click here" or "more")
  • Ensure adequate color contrast between link text and background
  • Distinguish between visited and unvisited links where appropriate
  • Keep link text reasonably short (generally under 100 characters)
  • Make sure links are accessible via keyboard navigation
  • Avoid using URLs as link text (unless the URL itself is important information)
  • Use absolute URLs for external links and relative URLs for internal links

Common Link Patterns

Navigation Menu

<nav>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="services.html">Services</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

Table of Contents

<div class="table-of-contents">
  <h3>Contents</h3>
  <ul>
    <li><a href="#section1">1. Introduction</a></li>
    <li><a href="#section2">2. Getting Started</a></li>
    <li><a href="#section3">3. Advanced Topics</a></li>
  </ul>
</div>

<section id="section1">
  <h2>1. Introduction</h2>
  <p>...</p>
</section>

Breadcrumb Navigation

<nav aria-label="Breadcrumb">
  <ul class="breadcrumbs">
    <li><a href="index.html">Home</a></li>
    <li><a href="products.html">Products</a></li>
    <li aria-current="page">Product Details</li>
  </ul>
</nav>
