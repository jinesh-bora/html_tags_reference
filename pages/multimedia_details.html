Images and Multimedia Tags

HTML elements for embedding images, video, audio, and other media

  • Back to Main Reference

About HTML Multimedia

Multimedia elements allow you to embed images, videos, audio, and other interactive content into your web pages. These elements enhance user experience by providing visual and auditory information and making your content more engaging.

HTML5 introduced native support for multimedia with the <video> and <audio> elements, reducing the need for third-party plugins like Flash. These elements, along with the <img> and <iframe> tags, form the core of HTML's multimedia capabilities.

Image Elements

<img>

Embeds an image into the document. It's an empty element (no closing tag).

Basic syntax: <img src="image-path.jpg" alt="Descriptive text">

The src attribute specifies the path to the image file, while the alt attribute provides alternative text for users who cannot view the image.

Example:

<img src="images/logo.png" alt="Company Logo" width="200" height="100">

Common Attributes:

  • src: Required. Specifies the path to the image
  • alt: Required. Provides alternative text for screen readers and displays if the image fails to load
  • width: Specifies the width of the image in pixels
  • height: Specifies the height of the image in pixels
  • loading: Specifies whether the browser should load the image immediately or defer loading (values: "eager", "lazy")
  • srcset: Specifies multiple image sources for different screen resolutions
  • sizes: Specifies image sizes for different viewport sizes (used with srcset)

<picture>

Provides a container for multiple image sources, allowing browsers to choose the most appropriate image based on device characteristics.

Example:

<picture>
  <source media="(min-width: 1200px)" srcset="large-image.jpg">
  <source media="(min-width: 600px)" srcset="medium-image.jpg">
  <img src="small-image.jpg" alt="Description of the image">
</picture>

The <picture> element contains <source> elements and one <img> element. The <img> element is required as a fallback.

<figure> and <figcaption>

Used to encapsulate media content with an optional caption.

Example:

<figure>
  <img src="chart.jpg" alt="Annual Sales Chart">
  <figcaption>Fig.1 - Annual Sales Growth 2022</figcaption>
</figure>

The <figure> element represents self-contained content, while <figcaption> provides a caption for the figure.

Video Elements

<video>

Embeds a video player into the document.

Basic syntax: <video src="video-file.mp4" controls></video>

Example with multiple sources for browser compatibility:

<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  <source src="movie.ogg" type="video/ogg">
  <p>Your browser does not support the video tag.</p>
</video>

Common Attributes:

  • src: Specifies the URL of the video file (can be omitted if using <source> elements)
  • controls: Adds video controls (play, pause, volume, etc.)
  • width and height: Specify dimensions of the video player
  • autoplay: Starts playing the video automatically (use with caution)
  • muted: Mutes the audio output
  • loop: Makes the video play repeatedly
  • poster: Specifies an image to be shown before the video plays
  • preload: Hints how much buffering is needed (values: "none", "metadata", "auto")
  • playsinline: Allows videos to play inline on iOS (instead of fullscreen)

<source>

Specifies multiple media resources for <video> and <audio> elements.

Example:

<source src="video.mp4" type="video/mp4">
<source src="video.webm" type="video/webm">

Browsers will use the first recognized format. The type attribute helps the browser determine if it can play the file without downloading it first.

<track>

Specifies text tracks (like subtitles or captions) for media elements.

Example:

<video controls>
  <source src="movie.mp4" type="video/mp4">
  <track src="subtitles_en.vtt" kind="subtitles" srclang="en" label="English">
  <track src="subtitles_es.vtt" kind="subtitles" srclang="es" label="Spanish">
</video>

The <track> element uses WebVTT (.vtt) files to provide timed text tracks.

Audio Elements

<audio>

Embeds sound content in a document.

Basic syntax: <audio src="audio-file.mp3" controls></audio>

Example with multiple sources:

<audio controls>
  <source src="song.mp3" type="audio/mpeg">
  <source src="song.ogg" type="audio/ogg">
  <p>Your browser does not support the audio element.</p>
</audio>

Common Attributes:

  • src: Specifies the URL of the audio file (can be omitted if using <source> elements)
  • controls: Adds audio controls (play, pause, volume, etc.)
  • autoplay: Starts playing the audio automatically (use with caution)
  • muted: Mutes the audio output
  • loop: Makes the audio play repeatedly
  • preload: Hints how much buffering is needed (values: "none", "metadata", "auto")

Embedded Content

<iframe>

Creates an inline frame that embeds another document within the current HTML document.

Example:

<iframe src="https://www.youtube.com/embed/VIDEO_ID" 
  width="560" height="315" 
  frameborder="0" 
  allowfullscreen>
</iframe>

Common uses for iframes include embedding YouTube videos, Google Maps, and other third-party content.

Common Attributes:

  • src: Specifies the URL of the document to embed
  • width and height: Specify dimensions of the iframe
  • frameborder: Specifies whether to display a border (0 for no border, 1 for border)
  • allowfullscreen: Allows the iframe to be displayed in fullscreen mode
  • loading: Specifies whether to load the iframe immediately or defer loading (values: "eager", "lazy")
  • sandbox: Applies extra restrictions to content in the iframe for security

Security Considerations:

Iframes can pose security risks through cross-site scripting. Best practices include:

  • Use the sandbox attribute to restrict capabilities
  • Use HTTPS sources when possible
  • Only embed content from trusted sources
  • Consider Content Security Policy (CSP) headers

Responsive Media Techniques

Responsive Images

There are several approaches to make images responsive:

1. Using CSS max-width

<img src="image.jpg" alt="Description" style="max-width:100%; height:auto;">

/* Or in your CSS file */
img {
  max-width: 100%;
  height: auto;
}

2. Using srcset for resolution switching

<img src="small.jpg"
     srcset="small.jpg 500w,
             medium.jpg 1000w,
             large.jpg 1500w"
     sizes="(max-width: 600px) 500px,
            (max-width: 1200px) 1000px,
            1500px"
     alt="Description">

The srcset attribute lists image files with their width descriptors (w), and sizes indicates the display size at different breakpoints.

3. Using the picture element for art direction

<picture>
  <source media="(min-width: 1200px)" srcset="landscape.jpg">
  <source media="(min-width: 600px)" srcset="square.jpg">
  <img src="portrait.jpg" alt="Description">
</picture>

This allows you to show completely different images based on screen size.

Responsive Video

Making videos responsive requires maintaining their aspect ratio:

1. Using CSS

<video controls width="100%">
  <source src="video.mp4" type="video/mp4">
</video>

/* Or for fixed aspect ratio */
.video-container {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 aspect ratio */
  height: 0;
  overflow: hidden;
}

.video-container iframe,
.video-container video {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}

2. Using the aspect-ratio property (modern CSS)

.video-container {
  width: 100%;
  aspect-ratio: 16 / 9;
}

Media Formats and Browser Support

Image Formats

Format Best Use Cases Browser Support
JPEG (.jpg, .jpeg) Photographs, complex images with many colors All browsers
PNG (.png) Images requiring transparency, screenshots, graphics All browsers
GIF (.gif) Simple animations, images with few colors All browsers
SVG (.svg) Logos, icons, illustrations that need to scale All modern browsers
WebP (.webp) Modern replacement for JPEG and PNG with better compression Most modern browsers except some older Safari versions
AVIF (.avif) Next-gen format with superior compression and quality Chrome, Firefox, partial support in Safari

Video Formats

Format File Extension MIME Type Browser Support
MP4 (H.264) .mp4 video/mp4 All browsers
WebM (VP8/VP9) .webm video/webm Most modern browsers
Ogg Theora .ogg, .ogv video/ogg Firefox, Chrome, Opera

Audio Formats

Format File Extension MIME Type Browser Support
MP3 .mp3 audio/mpeg All browsers
WAV .wav audio/wav Most browsers except older IE
Ogg Vorbis .ogg audio/ogg Firefox, Chrome, Opera

Accessibility Considerations

Image Accessibility

  • Alt text: Always provide meaningful alt text for images that convey information 
    <img src="chart.jpg" alt="Sales growth chart showing 15% increase in Q4 2024">
  • Decorative images: Use empty alt attributes for decorative images 
    <img src="decorative-divider.png" alt="">
  • Complex images: Provide detailed descriptions in context or using aria-describedby 
    <img src="complex-diagram.jpg" alt="Network diagram" aria-describedby="diagram-desc">
<div id="diagram-desc" class="visually-hidden">
  Detailed description of the diagram...
</div>

Video and Audio Accessibility

  • Captions and subtitles: Provide text alternatives for audio content 
    <video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="captions" src="captions.vtt" srclang="en" label="English" default>
</video>
  • Transcripts: Provide text transcripts for audio and video content
  • Audio descriptions: For videos, provide descriptions of visual content
  • Accessible controls: Ensure media players have keyboard-accessible controls
  • No autoplay: Avoid autoplaying content with audio, which can interfere with screen readers

Best Practices for Multimedia

  • Optimize file sizes: Compress images and media files for web use to improve loading times
  • Specify dimensions: Always include width and height attributes for images to prevent layout shifts
  • Use responsive techniques: Implement responsive design for all media to ensure it works on all devices
  • Provide fallbacks: Include fallback content for browsers that don't support specific features
  • Consider bandwidth: Use techniques like lazy loading for non-critical media
  • Accessibility: Always provide alternatives for users who cannot access the media
  • Format selection: Choose the right format for each use case (e.g., SVG for logos, WebP for photos)
  • Multiple sources: Provide multiple formats with the <source> element for maximum compatibility
