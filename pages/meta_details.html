Meta Tags

HTML elements that provide metadata about a document

  • Back to Main Reference

About HTML Metadata

Metadata is data about data—in the context of HTML, metadata provides information about the HTML document itself rather than the visible content. This metadata is not displayed on the page but is machine-parsable and used by browsers, search engines, and other web services.

HTML metadata is primarily defined using <meta> tags in the document's <head> section, although other elements like <title>, <link>, and <style> also contribute to a document's metadata.

The Meta Element

<meta>

Provides metadata about the HTML document. It's always placed within the <head> element.

The <meta> element is an empty element (it doesn't have a closing tag) and uses attributes to specify the type and content of metadata.

Key Attributes:

  • name: Specifies the type of metadata
  • content: Specifies the metadata value
  • charset: Specifies the character encoding for the document
  • http-equiv: Provides an HTTP header for the information/value in the content attribute
  • property: Used for social media metadata (e.g., Open Graph Protocol)

Basic Examples:

<!-- Character encoding -->
<meta charset="UTF-8">

<!-- Page description -->
<meta name="description" content="A comprehensive guide to HTML meta tags">

<!-- Viewport settings for responsive design -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- HTTP-equiv example for refresh -->
<meta http-equiv="refresh" content="30">

Character Encoding

<meta charset>

Specifies the character encoding used in the document. This meta tag should be placed as early as possible within the <head> element, ideally as the first element after the opening <head> tag.

Example:

<meta charset="UTF-8">

Common character encodings include:

  • UTF-8: The most widely used encoding that supports virtually all characters and symbols (recommended)
  • ISO-8859-1: Latin alphabet encoding (less common now)
  • Windows-1252: Microsoft Windows Western European encoding (less common now)

Best practice is to always use UTF-8 for all HTML documents, as it supports all languages and a wide range of symbols.

Viewport Settings

<meta name="viewport">

Controls how a webpage is displayed on mobile devices. This is crucial for responsive web design.

Example:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Common viewport settings in the content attribute:

  • width=device-width: Sets the width of the page to follow the screen width of the device
  • initial-scale=1.0: Sets the initial zoom level when the page is first loaded
  • minimum-scale=1.0: Minimum zoom level allowed
  • maximum-scale=5.0: Maximum zoom level allowed
  • user-scalable=yes: Allows (yes) or disables (no) zoom in and out actions on the page

Example with additional parameters:

<meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1.0, maximum-scale=5.0, user-scalable=yes">

Note: Disabling user scaling (user-scalable=no) is generally considered a bad practice for accessibility. Users with visual impairments often rely on the ability to zoom in on content.

SEO Meta Tags

<meta name="description">

Provides a brief description of the webpage. Search engines often display this in search results.

Example:

<meta name="description" content="Learn about HTML meta tags, their attributes, and best practices for improving SEO and social media sharing.">

Best practices:

  • Keep descriptions between 150-160 characters (search engines may truncate longer descriptions)
  • Make them unique for each page
  • Accurately summarize the page content
  • Include relevant keywords naturally
  • Make it compelling to encourage clicks

<meta name="keywords">

Lists keywords relevant to the page content. While most major search engines now give this tag little to no weight, some smaller search engines might still use it.

Example:

<meta name="keywords" content="HTML, meta tags, SEO, web development, HTML5, metadata">

Note that keyword stuffing (excessive use of keywords) can be seen as a spammy practice and might negatively affect SEO.

<meta name="author">

Specifies the author of the webpage.

Example:

<meta name="author" content="John Doe">

<meta name="robots">

Directs search engine crawlers on how to index and follow links on the page.

Example:

<meta name="robots" content="index, follow">

Common values for the content attribute:

  • index: Allows search engines to index the page
  • noindex: Prevents search engines from indexing the page
  • follow: Allows search engines to follow links on the page
  • nofollow: Prevents search engines from following links on the page
  • noarchive: Prevents search engines from showing cached versions of the page
  • all: Equivalent to index, follow
  • none: Equivalent to noindex, nofollow

For specific search engines, you can use:

<meta name="googlebot" content="noindex, nofollow">
<meta name="bingbot" content="noindex, nofollow">

HTTP-Equiv Meta Tags

<meta http-equiv>

Provides an HTTP header for the information/value in the content attribute. These tags act as though they were HTTP response headers.

Common HTTP-Equiv Values:

1. Refresh

Redirects to another URL after a specified time interval or immediately refreshes the page.

<!-- Refresh the page every 30 seconds -->
<meta http-equiv="refresh" content="30">

<!-- Redirect to another page after 5 seconds -->
<meta http-equiv="refresh" content="5; url=https://example.com/new-page">

Note: Automatic page refreshes can disrupt the user experience and may pose accessibility issues.

2. Content-Type

Specifies the document's MIME type and character encoding (legacy approach, using <meta charset> is now preferred).

<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
3. X-UA-Compatible

Used for Internet Explorer compatibility mode.

<meta http-equiv="X-UA-Compatible" content="IE=edge">

This tells Internet Explorer to use the highest mode available.

4. Content-Security-Policy

Helps prevent Cross-Site Scripting (XSS) and data injection attacks.

<meta http-equiv="Content-Security-Policy" content="default-src 'self'">

This example restricts resources to the same origin as the page itself.

5. Cache-Control

Controls caching behavior.

<meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate">
<meta http-equiv="Pragma" content="no-cache">
<meta http-equiv="Expires" content="0">

This prevents caching of the page.

Social Media Meta Tags

Open Graph Protocol

Enables any web page to become a rich object in social media platforms like Facebook, LinkedIn, and Pinterest.

Open Graph meta tags use the property attribute instead of name.

Basic Open Graph example:

<meta property="og:title" content="HTML Meta Tags Reference">
<meta property="og:description" content="A comprehensive guide to HTML meta tags and their usage.">
<meta property="og:image" content="https://example.com/image.jpg">
<meta property="og:url" content="https://example.com/meta-tags">
<meta property="og:type" content="website">
<meta property="og:site_name" content="HTML Reference">

Common Open Graph Properties:

  • og:title: The title of the page
  • og:description: A brief description of the content
  • og:image: The URL of an image to represent the content
  • og:url: The canonical URL of the page
  • og:type: The type of content (e.g., website, article, product)
  • og:site_name: The name of the overall website
  • og:locale: The locale of the content (e.g., en_US)

For articles, you can add:

<meta property="article:published_time" content="2025-06-04T12:00:00+00:00">
<meta property="article:author" content="John Doe">
<meta property="article:section" content="Web Development">
<meta property="article:tag" content="HTML">
<meta property="article:tag" content="Meta Tags">

Twitter Cards

Enables rich media experiences when links are shared on Twitter.

Basic Twitter Card example:

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:site" content="@yourusername">
<meta name="twitter:title" content="HTML Meta Tags Reference">
<meta name="twitter:description" content="A comprehensive guide to HTML meta tags and their usage.">
<meta name="twitter:image" content="https://example.com/image.jpg">

Common Twitter Card Properties:

  • twitter:card: The type of card (summary, summary_large_image, app, player)
  • twitter:site: The Twitter @username of the website
  • twitter:creator: The Twitter @username of the content creator
  • twitter:title: The title of the content
  • twitter:description: A brief description of the content
  • twitter:image: The URL of an image to represent the content
  • twitter:image:alt: A text description of the image for accessibility

Link Tags

<link>

Defines the relationship between the current document and an external resource. Most commonly used to link to stylesheets, but also used for favicons, alternate versions of a page, and more.

Key Attributes:

  • rel: Required. Specifies the relationship between the current document and the linked document
  • href: Specifies the location of the linked document
  • type: Specifies the media type of the linked document
  • media: Specifies what media/device the linked document is optimized for
  • sizes: Specifies the size of icons for visual media
  • hreflang: Specifies the language of the text in the linked document

Common Uses:

1. Stylesheets
<!-- Main stylesheet -->
<link rel="stylesheet" href="styles.css">

<!-- Media-specific stylesheet -->
<link rel="stylesheet" href="print.css" media="print">

<!-- Preload critical CSS -->
<link rel="preload" href="critical.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
2. Favicons and App Icons
<!-- Basic favicon -->
<link rel="icon" href="favicon.ico">

<!-- Modern favicon with size specification -->
<link rel="icon" type="image/png" sizes="32x32" href="favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="favicon-16x16.png">

<!-- Apple Touch Icon -->
<link rel="apple-touch-icon" sizes="180x180" href="apple-touch-icon.png">

<!-- Android Chrome icons -->
<link rel="manifest" href="site.webmanifest">
3. Alternative Page Versions
<!-- Canonical URL -->
<link rel="canonical" href="https://example.com/canonical-page">

<!-- Alternate language versions -->
<link rel="alternate" hreflang="es" href="https://example.com/es/page">
<link rel="alternate" hreflang="fr" href="https://example.com/fr/page">

<!-- RSS/Atom feeds -->
<link rel="alternate" type="application/rss+xml" title="RSS Feed" href="/feed.xml">
4. Preconnect, Prefetch, and Preload
<!-- Preconnect to important third-party origins -->
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

<!-- Prefetch likely next page -->
<link rel="prefetch" href="next-page.html">

<!-- Preload critical resources -->
<link rel="preload" href="critical-font.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="hero-image.jpg" as="image">

Mobile and App Meta Tags

Mobile Theme Color

Specifies the color of the browser UI elements, like the address bar in mobile browsers.

<meta name="theme-color" content="#4285f4">

Apple Specific Meta Tags

<!-- Hide Safari UI components -->
<meta name="apple-mobile-web-app-capable" content="yes">

<!-- Status bar appearance -->
<meta name="apple-mobile-web-app-status-bar-style" content="black">

<!-- App title -->
<meta name="apple-mobile-web-app-title" content="My App">

<!-- Prevent automatic detection of phone numbers -->
<meta name="format-detection" content="telephone=no">

Microsoft Tile Tags

<meta name="msapplication-TileColor" content="#2b5797">
<meta name="msapplication-TileImage" content="/mstile-144x144.png">
<meta name="msapplication-config" content="/browserconfig.xml">

Complete Meta Tags Example

<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Character encoding -->
    <meta charset="UTF-8">
    
    <!-- Viewport settings for responsive design -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- IE compatibility mode -->
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    
    <!-- Page title (displayed in browser tabs and bookmarks) -->
    <title>Comprehensive Meta Tags Example</title>
    
    <!-- SEO meta tags -->
    <meta name="description" content="A comprehensive example of HTML meta tags and their usage for SEO, social media, and more.">
    <meta name="keywords" content="HTML, meta tags, SEO, web development">
    <meta name="author" content="John Doe">
    <meta name="robots" content="index, follow">
    
    <!-- Canonical URL -->
    <link rel="canonical" href="https://example.com/meta-tags">
    
    <!-- Open Graph meta tags for social media -->
    <meta property="og:title" content="Comprehensive Meta Tags Example">
    <meta property="og:description" content="A comprehensive example of HTML meta tags and their usage for SEO, social media, and more.">
    <meta property="og:image" content="https://example.com/image.jpg">
    <meta property="og:url" content="https://example.com/meta-tags">
    <meta property="og:type" content="website">
    <meta property="og:site_name" content="HTML Reference">
    
    <!-- Twitter Card meta tags -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:site" content="@yourusername">
    <meta name="twitter:title" content="Comprehensive Meta Tags Example">
    <meta name="twitter:description" content="A comprehensive example of HTML meta tags and their usage for SEO, social media, and more.">
    <meta name="twitter:image" content="https://example.com/image.jpg">
    
    <!-- Favicon and app icons -->
    <link rel="icon" href="favicon.ico">
    <link rel="icon" type="image/png" sizes="32x32" href="favicon-32x32.png">
    <link rel="icon" type="image/png" sizes="16x16" href="favicon-16x16.png">
    <link rel="apple-touch-icon" sizes="180x180" href="apple-touch-icon.png">
    <link rel="manifest" href="site.webmanifest">
    
    <!-- Mobile theme color -->
    <meta name="theme-color" content="#4285f4">
    
    <!-- Stylesheets -->
    <link rel="stylesheet" href="styles.css">
    
    <!-- Preload critical resources -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link rel="preload" href="critical.css" as="style">
    
    <!-- Alternate versions -->
    <link rel="alternate" hreflang="es" href="https://example.com/es/meta-tags">
    <link rel="alternate" type="application/rss+xml" title="RSS Feed" href="/feed.xml">
</head>
<body>
    <!-- Page content... -->
</body>
</html>

Best Practices for Meta Tags

  • Prioritize important meta tags: Focus on charset, viewport, title, description, canonical URL, and social media tags
  • Use UTF-8 encoding: Always include <meta charset="UTF-8"> as the first meta tag in the head
  • Set a proper viewport: Include the viewport meta tag for responsive design
  • Write compelling descriptions: Create unique, accurate, and engaging meta descriptions (150-160 characters)
  • Be mindful of social sharing: Include Open Graph and Twitter Card meta tags
  • Use canonical URLs: Always specify the canonical URL to prevent duplicate content issues
  • Keep titles under 60 characters: Ensure your title tags are concise but descriptive
  • Optimize images for social sharing: Use appropriately sized images for social media (typically 1200x630 pixels)
  • Specify language and direction: Use the lang attribute on the <html> element
  • Include favicons and app icons: Provide appropriate icons for different platforms and devices
  • Test your meta tags: Use validation tools to ensure your tags are working correctly
  • Avoid meta keyword stuffing: If you use the keywords meta tag, keep it focused and relevant
  • Implement resource hints: Use preconnect, prefetch, and preload where appropriate to improve performance
